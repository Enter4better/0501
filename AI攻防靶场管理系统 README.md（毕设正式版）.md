# AI攻防靶场管理系统 README\.md（毕设正式版）

# 一、项目概述

本项目为人工智能与网络安全方向毕业设计，设计并实现了一套基于Vue3、Flask与Docker技术栈的智能攻防靶场管理系统。系统以“轻量化部署、可视化操作、智能化攻防”为核心目标，提供靶场环境快速部署、历史靶场管理、多类型攻击模拟、智能实时防御、网络拓扑可视化、全局日志监控、用户管理及watchdog实时日志监听、异步队列通信等功能，可作为网络安全实训、攻防对抗演练的基础平台，也可用于相关教学演示，具有较强的实用性和可扩展性。

# 三、技术栈详解

## 3\.1 前端技术栈

- **Vue 3 \+ Vite**：前端框架核心，Vite作为构建工具，实现快速开发、热更新，提升开发效率，适配现代前端开发需求。

- **Element Plus**：基于Vue 3的UI组件库，提供统一、美观的界面组件，简化页面开发，保证界面的一致性和专业性，支撑登录注册、历史靶场管理等新增页面的开发。

- **Vue Router**：前端路由管理工具，实现页面之间的无刷新跳转，新增登录注册路由、历史靶场详情路由，规划系统路由结构，提升用户体验。

- **Axios**：HTTP请求封装工具，实现前端与后端的接口通信，处理请求拦截、响应拦截（如登录态验证），保证数据交互的稳定性。

- **@antv/g6**：可视化图表库，用于实现网络拓扑图的绘制、拖拽、缩放等功能，直观展示靶场网络结构。

- **动态动画插件**：新增攻防动画效果，实现攻击、防御过程的实时可视化展示，增强交互体验。

## 3\.2 后端技术栈

- **Python 3**：后端开发语言，语法简洁、生态完善，适合快速开发后端接口，支持异步编程特性。

- **Flask**：轻量级Python Web框架，用于构建后端API服务，新增用户管理、历史靶场管理、数据库操作、靶场创建日志记录等接口，路由配置灵活，易于扩展。

- **Flask\-CORS**：跨域资源共享插件，解决前端与后端跨域访问问题，保证接口正常通信。

- **Docker SDK for Python**：Docker官方Python SDK，用于通过代码控制Docker容器的创建、启动、停止、删除等操作，实现靶场环境的自动化部署，支撑历史靶场的管理功能。

- **PyMySQL**：Python操作MySQL数据库的工具，实现用户信息、靶场数据、日志数据（含靶场创建日志）的持久化存储与查询。

- **密码加密工具**：实现用户密码加密存储，保障用户信息安全，支撑登录注册功能。

- **Watchdog**：Python日志监听工具，用于实时监听多方日志（靶场容器日志、系统攻防日志、数据库操作日志），捕捉日志变化并实时反馈。

- **Asyncio \+ 异步队列**：基于Python异步编程框架，实现异步队列通信，处理Watchdog监听的日志数据、攻防请求等任务，避免阻塞系统主线程，提升系统并发性能和响应速度。

# 四、系统需求分析

## 4\.1 功能需求

系统围绕“用户管理\-靶场管理\-攻击模拟\-防御拦截\-监控分析”的核心流程，设计以下功能模块，满足攻防演练及教学实训需求，新增功能已明确标注：

1. **用户管理模块**：实现用户注册、登录功能，登录后右上角显示用户名/头像，悬停展示用户详细信息；支持用户信息的基础管理，保障系统访问权限可控。

2. **靶场环境管理模块**：支持Docker镜像选择、端口映射配置、容器名称设置，实现靶场的一键创建、单个删除、全部清理，实时反馈操作结果；新增“已有靶场”管理功能，可查看所有已创建未删除的历史靶场，支持点击进入靶场进行攻防演练、删除靶场、修改靶场信息（如镜像、端口、名称）。

3. **攻击模拟模块**：支持多种攻击类型（在原有SQL注入、XSS、命令执行基础上拓展），有攻防动态动画；攻击与防御实时轮替进行，攻击模块检测到攻击被成功防御后，自动发动下一轮升级攻击，直至该类型最高级别攻击被成功阻止。

4. **智能防御模块**：防御功能默认常启，无需手动开启；当攻击发生时，防御系统立即启动拦截，同步展示防御动画，与攻击模块形成实时轮替交互，持续抵御升级后的攻击。

5. **网络拓扑模块**：可视化展示靶场与本机的网络连接关系，支持拓扑图的拖拽、缩放，实时更新靶场运行状态；进入历史靶场时，同步加载该靶场的网络拓扑信息。

6. **日志监控模块**：汇总攻击日志、防御日志、靶场日志（记录靶场创建时间、创建人、靶场配置等信息）；所有日志持久化存储至数据库，支持按时间、类型、靶场等条件查询，便于追溯攻防过程和靶场操作记录，分析系统运行状态。

7. **AI决策分析模块**：基于攻防数据，生成智能分析报告，总结攻击类型、防御效果、攻击升级过程，提供针对性防御建议。

8. **数据库模块**：接入MySQL数据库，实现用户信息、历史靶场数据、所有日志数据的持久化存储，解决原有内存临时存储日志丢失的问题，支持数据的长期留存和查询。

9. **Watchdog实时日志监听模块（新增）**：通过Watchdog工具实时监听多方日志，包括靶场容器运行日志、系统攻防操作日志、数据库操作日志、用户操作日志，实现日志变化的实时捕捉，确保日志数据不遗漏、实时同步至日志监控模块。

10. **异步队列通信模块（新增）**：基于Asyncio实现异步队列，承接Watchdog监听的日志数据、攻防请求任务、靶场操作任务等，实现任务的异步处理，避免主线程阻塞，提升系统并发能力和响应速度；确保多方日志数据、攻防任务高效通信，实现日志实时展示、攻防过程流畅衔接。

## 4\.2 非功能需求

- **易用性**：界面设计简洁直观，操作流程清晰，登录注册流程便捷，历史靶场管理、攻防操作简单易懂，无需复杂配置，用户可快速上手。

- **稳定性**：保证靶场创建/删除/修改、攻击发起、防御拦截、用户登录注册、Watchdog日志监听、异步队列通信等核心功能稳定运行，无异常崩溃，日志记录完整，数据库连接稳定。

- **可视化**：通过拓扑图、日志列表、攻防动画等形式，直观展示系统运行状态、靶场信息和攻防过程，Watchdog监听的日志实时同步展示，提升用户体验。

- **可扩展性**：系统架构设计合理，支持后续新增攻击类型、防御策略、靶场模板、用户权限分级等功能，同时支持Watchdog监听范围拓展、异步队列任务类型拓展。

- **安全性**：用户密码加密存储，登录态验证严格，防止未授权访问；数据库数据定期备份，保障数据安全；Watchdog监听日志可设置权限，防止日志数据泄露。

- **高效性**：通过异步队列通信减少系统阻塞，提升任务处理效率；Watchdog日志监听延迟低，确保日志实时同步，攻防过程、靶场操作无明显卡顿。

# 五、系统总体设计

## 5\.1 系统架构设计

系统采用前后端分离架构，在原有四层结构基础上新增数据库层，优化各层逻辑，新增Watchdog日志监听子层和异步队列通信子层，形成前端展示层、后端服务层、数据存储层、靶场环境层、数据库层五层结构，各层独立运行、相互通信，确保系统的灵活性和可维护性。

1. **前端展示层**：基于Vue3构建登录注册页面、历史靶场管理页面、靶场信息修改页面，添加攻防动画组件；负责用户界面展示、用户操作交互、数据可视化，通过Axios与后端接口通信，获取数据并渲染页面，实现登录态的本地存储与展示，同步展示Watchdog监听的实时日志。

2. **后端服务层**：基于Flask构建，新增用户管理、历史靶场管理、数据库操作、靶场创建日志记录、Watchdog日志监听、异步队列通信等接口；处理前端请求，调用Docker SDK控制靶场容器，实现攻击、防御逻辑（优化攻防轮替机制），记录系统日志并同步至数据库，实现登录态验证和权限控制；集成Watchdog工具和异步队列，处理日志监听和任务通信。

3. **数据存储层**：基于MySQL数据库，替代原有内存临时存储，实现用户信息、历史靶场数据、攻击日志、防御日志、靶场创建日志的持久化存储，支持数据的查询、修改、删除操作；同步存储Watchdog监听的所有日志数据，确保日志可追溯。

4. **靶场环境层**：基于Docker容器技术，通过后端接口动态创建、管理靶场容器，支持历史靶场的重新启动、信息修改和删除，模拟真实网络环境，为攻防演练提供基础；Docker镜像优先选用Docker官方硬化镜像（Docker Hardened Images），提升靶场环境安全性，减少漏洞风险；靶场容器运行日志实时被Watchdog监听。

5. **数据库层**：采用MySQL数据库，设计用户表、靶场表、日志表（含攻击、防御、靶场创建日志），实现数据的结构化存储，保障数据的一致性和可追溯性，支持数据备份与恢复；数据库操作日志被Watchdog实时监听，确保操作可追溯。

6. **Watchdog日志监听子层（新增）**：部署Watchdog监听服务，实时监听靶场容器日志、系统攻防日志、数据库操作日志、用户操作日志，捕捉日志新增、修改、删除等变化，将日志数据实时推送至异步队列。

7. **异步队列通信子层（新增）**：基于Asyncio实现异步队列，接收Watchdog推送的日志数据、前端发起的攻防任务、靶场操作任务等，异步处理任务并反馈结果；实现后端各模块间的高效通信，避免主线程阻塞，提升系统并发性能。

# 六、系统部署与运行步骤

本系统部署流程简单，新增数据库部署、Watchdog日志监听及异步队列配置步骤，无需复杂配置，按照以下步骤即可完成部署并运行，适合毕业设计演示和本地测试。

## 6\.1 前置准备

1. 安装Docker：前往Docker官方网站（https://www\.docker\.com/）下载对应系统版本的Docker，安装完成后启动Docker服务（确保Docker正常运行）；建议优先使用Docker官方硬化镜像（Docker Hardened Images），提升靶场环境安全性，减少漏洞风险。

2. 安装Python：下载Python 3\.7及以上版本，安装时勾选“Add Python to PATH”，完成后通过命令行输入`python \-\-version`验证安装成功。

3. 安装Node\.js：下载Node\.js 16及以上版本，安装完成后通过命令行输入`node \-\-version`验证安装成功。

4. 安装MySQL：下载MySQL 8\.0及以上版本，安装完成后启动MySQL服务，创建数据库（建议数据库名：cyber\_range），记录数据库用户名和密码（用于后续配置）。

5. 下载项目源码：将本项目的backend（后端）和frontend（前端）两个文件夹保存至本地同一目录下，建议目录结构如下：

```plain text
AI攻防靶场管理系统/
├── backend/                # 后端服务文件夹
│   ├── config.py           # 数据库配置文件（需修改）
│   ├── app.py             # Flask 主接口
│   ├── run.py             # 启动入口
│   ├── watchdog_monitor.py # Watchdog日志监听服务
│   ├── async_queue.py     # 异步队列通信服务
│   └── requirements.txt   # 依赖
└── frontend/               # 前端项目文件夹
```

6\. 配置数据库：进入backend文件夹，找到config\.py文件，修改数据库配置（数据库地址、用户名、密码、数据库名），确保后端能够正常连接MySQL数据库。

7\. 配置Watchdog与异步队列：进入backend文件夹，找到watchdog\_monitor\.py和async\_queue\.py文件，根据实际需求配置日志监听路径（如靶场容器日志路径、系统日志路径）、异步队列参数（如队列大小、任务超时时间），确保监听正常、通信流畅。

## 6\.2 后端启动步骤

1. 打开命令行，进入backend文件夹：
        `cd backend`

2. 安装后端依赖包（依赖清单见requirements\.txt，包含PyMySQL、watchdog、asyncio等相关依赖）：
        `pip install \-r requirements\.txt`

3. 初始化数据库：执行数据库迁移命令（具体命令见backend文件夹下的readme说明），创建用户表、靶场表、日志表等必要数据表。

4. 启动异步队列服务：
       `python async\_queue\.py`（保持该命令行窗口开启，不可关闭）
      

5. 打开新的命令行，进入backend文件夹，启动Watchdog日志监听服务：
        `python watchdog\_monitor\.py`（保持该命令行窗口开启，不可关闭）
      

6. 打开新的命令行，进入backend文件夹，启动后端主服务：
        `python run\.py`

7. 验证后端启动成功：命令行提示“\* Running on http://0\.0\.0\.0:5000”，表示后端服务正常运行，默认端口为5000；同时确认数据库连接成功、Watchdog监听服务正常、异步队列服务正常，无报错信息。

## 6\.3 前端启动步骤

1. 打开新的命令行，进入frontend文件夹：
       `cd frontend`

2. 安装前端依赖包：
       `npm install`
        注：若安装速度较慢，可使用npm镜像源加速，如：`npm install \-\-registry=https://registry\.npm\.taobao\.org`

3. 启动前端服务：`npm run dev`

4. 验证前端启动成功：命令行提示“VITE vx\.x\.x ready in x ms”，并给出访问地址（默认http://127\.0\.0\.1:3000）。

## 6\.4 系统访问

1. 打开浏览器，输入前端访问地址（http://127\.0\.0\.1:3000），首先进入登录页面，点击“注册”按钮完成用户注册（注册信息将存储至MySQL数据库）。

2. 使用注册的账号密码登录系统，登录后右上角显示用户名/头像，悬停可查看用户详细信息。

3. 登录后即可进入系统首页，完成靶场创建、历史靶场管理、攻防演练等各项功能操作；同时可在日志面板查看Watchdog实时监听的所有日志，实现日志实时监控。

# 七、核心功能操作说明

## 7\.1 用户登录与注册（新增）

1. 访问系统首页，自动跳转至登录页面，登录页面采用科技风动态设计，包含账号、密码输入框及“登录”“注册”按钮。

2. 点击“注册”按钮，进入注册页面，输入用户名、密码、确认密码等信息，提交后完成注册，注册信息将加密存储至数据库。

3. 使用注册的账号密码登录系统，登录成功后跳转至系统首页，右上角显示用户名（或头像），悬停用户名/头像，将展示用户ID、注册时间等详细信息。

## 7\.2 靶场环境管理（优化）

1. 进入“生成靶场”页面（/env），配置Docker镜像（默认nginx:latest，建议选用Docker硬化镜像）、端口映射（默认8080:80）、容器名称（默认cyber\_target），点击“创建靶场”按钮。

2. 靶场创建成功后，系统自动记录靶场创建日志（包含创建时间、创建人、靶场配置等），并同步存储至数据库；同时在“已有靶场”列表中新增该靶场记录；Watchdog实时监听靶场容器启动日志，同步推送至日志监控模块。

3. 进入“已有靶场”管理页面，可查看所有已创建未删除的历史靶场，列表展示靶场名称、镜像、端口、创建时间等信息。

4. 点击历史靶场条目，可进入该靶场的攻防演练页面，开展攻击模拟与防御操作；点击“删除”按钮，可删除该历史靶场（同时删除对应的容器和数据库记录）；点击“修改”按钮，可修改靶场的镜像、端口、名称等信息，并同步更新数据库；上述操作日志均被Watchdog实时监听并记录。

5. 点击“删除当前”按钮，可删除当前运行的靶场容器；点击“清理全部”按钮，可删除所有与靶场相关的容器及历史靶场记录（同步删除数据库中的靶场数据和相关日志）。

## 7\.3 攻击模拟与智能防御（优化）

1. 进入“攻击面板”页面（/attack）或历史靶场的攻防演练页面，选择攻击类型（支持多种类型，如SQL注入、XSS、命令执行等），输入目标端口（默认8080）。

2. 点击“发起攻击”按钮，系统启动攻击，同时展示攻击动态动画；此时防御功能已默认常启，立即启动拦截，展示防御动画，攻防过程实时轮替进行；攻击、防御操作日志通过异步队列实时推送至日志监控模块，Watchdog同步监听并记录。

3. 当防御系统成功拦截当前攻击后，攻击模块自动检测拦截结果，立即发动下一轮升级攻击（攻击强度、复杂度提升），防御系统同步升级防御策略，持续拦截；异步队列负责处理攻防任务的调度，避免阻塞，确保攻防过程流畅。

4. 重复步骤3，直至该类型攻击升级至最高级别，且被防御系统成功阻止，攻击模块停止攻击，系统记录完整的攻击升级过程和防御结果至日志（同步存储至数据库）；所有日志均被Watchdog实时监听，确保无遗漏。

5. 攻防过程中，可实时查看攻击日志和防御日志，同步观察攻防动画，直观了解攻击与防御的交互过程。

## 7\.4 网络拓扑查看

1. 进入“网络拓扑”页面（/topo），系统将自动加载当前运行靶场与本机的网络连接关系；进入历史靶场演练页面，将加载该历史靶场的网络拓扑信息。

2. 可通过鼠标拖拽拓扑图节点、滚轮缩放，查看详细的网络结构和端口映射信息。

3. 当靶场容器创建、删除、修改端口时，拓扑图将实时更新，反映当前靶场运行状态；相关操作日志被Watchdog实时监听并同步至日志面板。

## 7\.5 日志监控与AI分析（优化）

1. 进入“日志面板”页面（/logs），可查看全局日志，包括攻击日志、防御日志、靶场创建日志及Watchdog实时监听的所有日志（靶场容器日志、数据库操作日志、用户操作日志），所有日志均持久化存储至数据库，支持按日志类型、时间范围、靶场名称等条件查询。

2. 日志详情包含操作人、操作时间、操作内容（如靶场创建配置、攻击类型、防御结果等），Watchdog监听的日志将标注“实时监听”标识，便于区分，可追溯攻防过程和靶场操作记录。

3. 在系统首页（控制台），点击“生成攻防分析”按钮，系统将基于当前攻防日志、攻击升级过程、防御结果，生成AI智能分析报告，提供攻击类型、防御效果及优化建议。

## 7\.6 Watchdog实时日志监听与异步队列通信（新增）

1. Watchdog日志监听：系统启动后，Watchdog服务自动运行，实时监听预设路径下的多方日志，包括靶场容器运行日志、系统攻防操作日志、数据库操作日志、用户操作日志，一旦检测到日志新增、修改，立即捕捉并推送至异步队列。

2. 异步队列通信：异步队列接收Watchdog推送的日志数据、前端发起的攻防任务、靶场操作任务等，异步处理任务，避免主线程阻塞；例如，攻防任务与日志监听任务并行处理，确保攻防过程流畅、日志实时同步。

3. 日志同步展示：Watchdog监听的日志通过异步队列快速推送至日志监控模块，在日志面板实时展示，延迟低、无遗漏，用户可实时查看系统所有操作和运行日志。

4. 异常监控：若Watchdog监听服务或异步队列出现异常，系统将自动记录异常日志，并在控制台给出提示，便于排查问题；可通过重启对应服务（watchdog\_monitor\.py、async\_queue\.py）恢复正常。

# 十一、注意事项

1. 运行系统前，必须确保Docker服务、MySQL服务已启动，同时需启动异步队列服务和Watchdog日志监听服务，否则无法创建靶场容器、无法连接数据库、无法实现日志实时监听，导致系统核心功能无法正常使用。

2. 首次创建靶场时，系统会自动拉取Docker镜像（如nginx:latest），镜像拉取速度取决于网络环境，请耐心等待；建议优先使用Docker硬化镜像，减少漏洞风险。

3. 部署后端前，需先修改backend/config\.py文件中的数据库配置，确保数据库地址、用户名、密码、数据库名正确，否则无法连接数据库；同时配置watchdog\_monitor\.py和async\_queue\.py中的参数，确保日志监听和异步通信正常。

4. 若出现端口冲突（如5000端口、3000端口、MySQL默认3306端口被占用），可修改后端run\.py中的端口、前端vite\.config\.js中的代理端口，或修改MySQL端口。

5. 用户注册时，建议设置复杂密码，保障账号安全；管理员需定期备份数据库，防止数据丢失。

6. 修改历史靶场信息后，需重启靶场容器才能生效；删除历史靶场时，将同步删除该靶场的容器、相关日志及数据库记录，操作前请确认。

7. 攻防演练过程中，若出现动画卡顿，可刷新页面或降低浏览器渲染压力；若攻击无法升级，需检查靶场容器是否正常运行、防御模块是否开启、异步队列服务是否正常。

8. 若Watchdog日志监听异常（如日志无法实时展示），需检查watchdog\_monitor\.py服务是否正常运行、监听路径是否正确；若异步队列阻塞，可重启async\_queue\.py服务，或调整队列参数。

9. 本系统仅用于教学、实训、毕业设计演示，请勿用于真实网络环境的攻击测试，避免违反相关法律法规。
